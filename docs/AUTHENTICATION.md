# Authentication Guide

MCP Email RW supports multiple authentication methods to connect to your email accounts securely.

## Supported Methods

| Method | Description | Use when |
|--------|-------------|----------|
| **Password** | Standard IMAP/SMTP login | Private or self-hosted servers |
| **App Password** | Provider-generated app-specific password | Gmail, Yahoo, iCloud with 2FA enabled |
| **OAuth2** | OAuth 2.0 with access and refresh tokens | Google Workspace, Microsoft 365 |

## Password Authentication

Basic username/password authentication. The email address is used as the username.

```json
{
  "auth": {
    "type": "password",
    "password": "your-email-password"
  }
}
```

> **Warning**: Plain password auth is the least secure option. Use app passwords or OAuth2 when available.

## App-Specific Passwords

App passwords are single-use passwords generated by your email provider specifically for third-party applications. They are the recommended method for providers that support them.

### Gmail

1. Go to [Google Account Security](https://myaccount.google.com/security)
2. Enable **2-Step Verification** if not already enabled
3. Go to [App Passwords](https://myaccount.google.com/apppasswords)
4. Select **Mail** and your device
5. Click **Generate** — copy the 16-character password
6. Use this password in your account setup

```json
{
  "auth": {
    "type": "app-password",
    "password": "xxxx xxxx xxxx xxxx"
  }
}
```

> **Note**: You must also enable IMAP in Gmail settings: Settings → See all settings → Forwarding and POP/IMAP → Enable IMAP.

### Yahoo

1. Go to [Yahoo Account Security](https://login.yahoo.com/account/security)
2. Enable **Two-step verification**
3. Click **Generate app password**
4. Select **Other App** and enter a name (e.g., "MCP Email")
5. Copy the generated password

### iCloud

1. Go to [Apple ID Account](https://appleid.apple.com/account/manage)
2. Sign in and go to **Sign-In and Security**
3. Select **App-Specific Passwords**
4. Click **Generate an app-specific password**
5. Enter a label (e.g., "MCP Email") and copy the password

> **Note**: Use your full iCloud email address (e.g., `user@icloud.com`) as the username.

## OAuth2 Authentication

OAuth2 provides the most secure authentication by using short-lived access tokens with automatic refresh.

### Google OAuth2 Setup

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create a new project or select an existing one
3. Enable the **Gmail API**
4. Go to **Credentials** → **Create Credentials** → **OAuth 2.0 Client ID**
5. Set application type to **Desktop app**
6. Note the **Client ID** and **Client Secret**
7. Configure the OAuth consent screen with required scopes:
   - `https://mail.google.com/`

```json
{
  "auth": {
    "type": "oauth2",
    "clientId": "your-client-id.apps.googleusercontent.com",
    "clientSecret": "your-client-secret",
    "refreshToken": "your-refresh-token",
    "accessToken": "your-access-token"
  }
}
```

To obtain refresh and access tokens, use the Google OAuth2 Playground or implement the OAuth2 flow.

### Microsoft / Outlook OAuth2 Setup

1. Go to [Azure Portal - App Registrations](https://portal.azure.com/#blade/Microsoft_AAD_RegisteredApps/ApplicationsListBlade)
2. Click **New registration**
3. Set a name and choose the appropriate account type
4. Add a redirect URI (e.g., `http://localhost:3000/callback`)
5. Go to **Certificates & secrets** → **New client secret**
6. Note the **Application (client) ID** and **Client Secret**
7. Configure API permissions:
   - `IMAP.AccessAsUser.All`
   - `SMTP.Send`
   - `offline_access`

```json
{
  "auth": {
    "type": "oauth2",
    "clientId": "your-azure-client-id",
    "clientSecret": "your-client-secret",
    "refreshToken": "your-refresh-token",
    "accessToken": "your-access-token"
  }
}
```

## How Credentials Are Stored

Credentials are stored in the local configuration file at `~/.mcp-email/accounts.json`. This file contains:

- Account metadata (name, email, provider)
- Server settings (IMAP/SMTP host, port, TLS)
- Authentication credentials (passwords, tokens)

### Environment variables

You can reference environment variables in your configuration for sensitive values. Set them in your shell or `.env` file:

```bash
export EMAIL_PASSWORD="your-password"
export OAUTH_CLIENT_SECRET="your-secret"
```

## Security Best Practices

1. **Prefer OAuth2** when available — tokens can be revoked without changing your password
2. **Use app passwords** over plain passwords — they can be revoked individually
3. **Never commit** `accounts.json` or `.env` files to version control
4. **Restrict file permissions** on your config:
   ```bash
   chmod 600 ~/.mcp-email/accounts.json
   ```
5. **Rotate credentials** periodically, especially app passwords
6. **Enable 2FA** on all email accounts
7. **Review connected apps** regularly in your provider's security settings
